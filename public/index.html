Das ultimative Kontrollzentrum für dein Homelab

ModuLab.app trennt den hochperformanten Core von sandboxed GitHub-Modulen. Vollständig konfiguriert über eine moderne Weboberfläche, geschützt durch AES-256-Verschlüsselung und nahtlos integriert mit deinem OIDC-Provider.

Ein moderner Kern für flexible Netze

Erfahre, warum ModuLab weit über eine einfache Link-Sammlung hinausgeht.

🔑

OIDC SSO & Dynamic Prefix

Nutze deinen bevorzugten IdP (PocketID, Keycloak, Authentik, Authelia). Unser Dynamic Prefix Gate kombiniert dein Präfix automatisch mit den Suffixen _Admin und _User für eine strikte Einlasskontrolle am Hard Gate.

👥

JIT Profiling & Warteschleife

Automatische Profil-Synchronisation beim Erst-Login (Name, Vorname, Nachname, E-Mail, Avatar). Neue berechtigte User landen in einer Onboarding-Warteschleife, bis sie von Admins freigeschaltet und Standorten zugewiesen werden.

🌐

Multi-Standort-Scoping

Verwalte physisch getrennte Netze (Zuhause, VPS/Cloud, Backup-Hosts) in einer zentralen Instanz. Der Core erzwingt auf Datenbankebene eine strikte Isolation der Widget-Daten anhand der zugewiesenen Standort-ID.

📦

Modul Place & Soft-Reload

Installiere Widgets und Launcher direkt aus öffentlichen oder privaten GitHub-Repositories (via personalisierte PATs) mit nur einem Klick. Module werden im laufenden Betrieb ohne Container-Neustart dynamisch geladen.

🔔

Echtzeit-SSE & Mail-Queue

Erhalte Push-Benachrichtigungen über Ereignisse (z. B. neue Logins oder unberechtigte Zugriffe) in Echtzeit via WebSockets/SSE, gekoppelt mit einer asynchronen, Valkey-gestützten Mail-Warteschlange.

⚡

High-Performance API-Cache

Unser Valkey In-Memory Cache agiert als unschlagbarer Schutzschild vor GitHub-Rate-Limits und sorgt für eine extrem geringe Latenz bei jedem Seitenaufruf – ideal für schwächere Hardware im Homelab.

Zero-Trust Homelab

Verschlüsselung auf Anwendungsebene & Isolation

Datensouveränität fängt beim Fundament an. ModuLab speichert alle sensitiven Daten (Access Tokens, Passwörter, API-Keys) standardmäßig über eine symmetrische AES-256-GCM Verschlüsselung. Ohne den Hauptschlüssel in der lokalen .env bleibt deine relationale PostgreSQL-Datenbank unlesbarer Zeichensalat.

✓

Modul-Sandboxing: Fremder Code von Drittanbieter-Modulen wird in einer isolierten Sandbox ausgeführt und hat keinen Zugriff auf die sensiblen Host-Umgebungsvariablen.

✓

Netzwerk-Splitting: Der Docker-Stack trennt den Proxy-Frontend-Traffic strikt vom Backend-Netzwerk. PostgreSQL und Valkey besitzen keine Ports nach außen.

✓

Container-Härtung: Der Container-Core läuft standardmäßig unprivilegiert als Non-Root-User mit einem Read-Only-Dateisystem.

Fragen zu ModuLab

Alles, was du über die Lizenzierung, OIDC und die Installation wissen musst.

Warum wurde die AGPLv3-Lizenz gewählt?

▼

Die AGPLv3 (GNU Affero General Public License v3) stellt sicher, dass der Quellcode des ModuLab-Cores für immer open-source und frei bleibt. Sie schützt das Projekt vor "SaaS-Trittbrettfahrern". Das bedeutet: Wer ModuLab modifiziert als gehosteten Netzwerk-Dienst (z. B. in der Cloud) anbietet, ist gesetzlich verpflichtet, seine Modifikationen ebenfalls als AGPLv3 offenzulegen.

Welche Identity Provider (IdPs) werden unterstützt?

▼

Da ModuLab vollständig auf dem standardisierten OpenID Connect (OIDC) Protokoll aufbaut, funktioniert es herstellerunabhängig mit jedem kompatiblen OIDC Identity Provider. Erfolgreich spezifiziert sind lokale Setups wie Authelia, Authentik, Keycloak und PocketID. Über standardisierte Claims werden Benutzername, Vorname, Nachname, E-Mail-Adresse und die Avatar-Bild-URL automatisch synchronisiert.

Gibt es bereits fertige Module für den Start?

▼

Ja! Im Core von ModuLab ist fest das offizielle Community-Repository verankert. Zur ersten Veröffentlichung werden die Module "App-Launcher" (zum bequemen Aufrufen deiner lokalen Docker-Dienste), "System-Status" (zur Live-Visualisierung von CPU, RAM und Festplattenspeicher über Valkey) sowie ein administrativer "GitHub-Repo-Manager" direkt bereitstehen.

Kann ich ModuLab auch ohne Internetverbindung nutzen?

▼

Absolut. Solange dein OIDC-Provider lokal erreichbar ist (z. B. auf deinem Server zu Hause), läuft das gesamte System vollständig offline. Die GitHub-API für den Modul Store ist das einzige Feature, das eine Internetverbindung benötigt, um neue Updates oder alternative Repositories abzufragen. Die installierten Module arbeiten danach komplett autark in deinem Intranet.
